1.1.0 Sept 20, 2016

  • Upgraded all 3rd party libraries to their latest versions.
  • Added grunt tasks.
  • Converted to Web Mercator base maps.
  • Database schema tweaks to cathode length, voltage and thymus category (#3).
  • Refactor/redesign the method tab (#2).
  • Fixed bug that allowed invalid numeric inputs to be submitted to the database (#17).
  • Added "Mouse wheel zooming" setting to settings dialog (#5).
  • Made the new station dialog map larger and synchronized its extent with that of the main maps (#9).
  • Added a subtle visual cue (bold) to rows that have additional data attached to them via the diet, tag, health and notes dialog (#12).

1.0.0 January 3, 2014

  • Fixed layout bug with grid scrolling. Closes #28662.
  • Fixed bug that prevented the more info dialog from showing in certain situations. Closes #28676.
  • Batch function now calculates '0' for rows if no weight was entered. Closes #28666.
  • Dropdowns have [other] option now that allows the user to create new options. We still need to decide what happens on the back end with this. Ref #22892.
  • Added basic test that verifies that data was stored in the database correctly. Closes #28508.

0.13.0 December 19, 2013

  • Fixed bug related to adding new passes. Ref #28578.
  • Added "Notes" tab and associated text area in the more info dialog. Ref #27683.
  • Expanded batch functionality to allow batching with no weights. Also reworked workflow to prevent the user from having to manually add blank rows before running a batch. Ref #27685.
  • Created separate buttons for the tabs within the more info dialog ("Diet", "Tag", "Health", "Notes") that open the dialog with that specific tab opened. These buttons are also disabled if there is no row selected in the grid below to re-enforce to the user that they are related to the selected row. Ref #27682.
  • Added ability to navigate the grids with keyboard arrow keys (up, down, left and right). Ref #27677.
  • The more info dialog is now pre-populated with data if there was data previously entered for the fish. Ref #26253.
  • Batch weights are now rounded to one decimal place. Ref #25990.
  • Rows that have weights that have manually been cleared out are now included in batch weights. Ref #28579.

0.12.0 December 6, 2013

  • Duplicate station names are now allowed. Unique ids (GUID's) are used to identify them behind the scenes. Ref #26528.
  • Map cursor now changes to a "cross hair" cursor when dropping points on the map. Ref #28525.
  • True/False drop downs have been changed to Yes/No. Ref #28523.
  • All fields, selected map points and grids are cleared when the event is submitted or the cancel button is pressed. Ref #28370.
  • Added dominant overstory and understory drop down values. Ref #28361.
  • After creating a new station, a corresponding point is added to the main map and selected. Ref #28360.
  • The selected station map marker now turns yellow.
  • Fixed validation bug with point definition boxes that was preventing them from turning red when invalid. Ref #28242.
  • Up to 20 results are now shown in the stream search drop down menu. This seems to be OK for the streams that I tested. Ref #27688.
  • Diet tab (more info dialog): Changed "Unidentified fish" to "Fish" and added a new field called "Fish Species" which has the same drop down as the "Species Code" field in the catch grid. Ref #27687.
  • All drop downs now preserve the description value (as apposed to the code value) excluding "Species Code" and "Length Type".
  • Minor layout/heading cleanup.
  • Fixed incorrect layout of grids introduced at 0.11.0.
  • Removed type-ahead for station box since they can now be selected by clicking on the map.

0.11.0 November 29, 2013

  • All numeric text inputs now show validation errors as the user types. Ref #26463
  • Fixed bug that was causing a cell value to be lost when navigating back to that cell. Ref #27678.
  • Fixed bug that was preventing the user from successfully selecting a drop-down option by clicking on it. Ref #27684.
  • Adds blue markers for existing stations on maps. Ref #20323.
  • Updated the order of the Location tab to put the map first, then the station box, then the stream reach. Ref #27679.
  • Moved main tab buttons and the submit/cancel buttons to the header. This makes them accessible at all time no matter where you are on the page. Ref #27680.

0.10.0 November 8, 2013

  • Added Habitat tab and wired to database.
  • Upgraded the look-and-feel library (Bootstrap) to version 3.
  • Upgraded comboboxes to work with Bootstrap v3.

0.9.0 September 17, 2013

  • Added stream search box to both maps.
  • Fixed bug related to switching raft/boat cathode type. (#26461)
  • Upgraded Leaflet to 0.6.0.

0.8.0 September 3, 2013

  • Fixed problems with data transfer to database. All data is now being submitted correctly to the database.
  • Reports are now archived on the client side in case of problems on the server.
  • Collection event dates are now submitted appropriately.

0.7.0 August 21, 2013

  • Re-ordered tag form controls. Ref #25992.
  • Added unit labels according to the changes made by Craig to the fields spreadsheet. Ref #26103.
  • Cathode diameter is now automatically cleared and disabled when "Boat" is selected as the Cathode type in the Boat/Raft section on the Method tab. Ref #24794.
  • Fixed label misalignment for backpacks, canoe/barges, and raft/boats. #26124.
  • Finished wiring up more info dialog.
  • Complete reports can now be submitted to the database (except the Habitat tab which hasn't been implemented yet).

0.6.0 July 18, 2013

  • Added more info dialog to catch tab grid (not wired up to submit data to database yet). Ref #25757.

0.5.0 July 9, 2013

  • Added Event Date field to location tab. Ref #23811.
  • On successful submit, stream length and additional notes fields are cleared. Ref #20634.
  • Method tab controls finalized and validating.
  • Catch tab grid built (not wired up to submit data to database yet).

0.4.0 Nov 20, 2012

  • New collection events can now be submitted successfully and are being stored in the test database along with the stations data.
  • Added error reporter to most errors so that they are logged and an email is sent to the developer.
  • Added values for EVENT_DATE, GEO_DEF, and EVENT_ID for the sampling event feature. These are all values that are automatically calculated behind the scenes and sent with the rest of the data to the server.
  • Added fields for stream length and additional location notes below the verify map. Stream length is automatically filled in when defining the location using the Start, Distance & Direction method.

0.3.0 Nov 15, 2012

  • You can now create new stations.
  • Moved the Coordinate type buttons to a settings dialog toggled by a button in the header.
  • Added stream type drop down to new station dialog. This is automatically populated with the domain values for this field in the geodatabase.
  • Defining a location by waterbody id now works.
  • Added "Settings" text to the settings dialog toggle button.

0.2.0 Oct 31, 2012

  • Start | Distance | Direction location definition now works.
  • ID location definition now works.
